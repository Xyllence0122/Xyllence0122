<h2 align="center">🚀 &nbsp;F E A T U R E D &nbsp;P R O J E C T S</h2>

<br/>

<details open>
<summary><b>&nbsp;🤖 &nbsp;Autonomous Mecanum Tracking Robot &nbsp;·&nbsp; Solo Project</b></summary>
<br/>

> Vision-guided omnidirectional robot built end-to-end: **Raspberry Pi 5 vision computing + Arduino Mega real-time PWM control**, open-sourced under GPL v3.0.

| | |
|:--|:--|
| **Stack** | Python · OpenCV · Raspberry Pi 5 · Arduino Mega · Serial (115200 baud) |
| **Scale** | Full perception → kinematics → actuation pipeline on embedded hardware |
| **Performance** | Real-time 3-DOF omnidirectional tracking via mecanum inverse kinematics |
| **Security** | Auto-reconnect serial protocol with fail-safe stop on link loss |
| **Impact** | Dual-mode: autonomous visual tracking + USB gamepad teleoperation |
| **Repository** | [Xyllence0122/Autonomous-Mecanum-Tracking-Robot](https://github.com/Xyllence0122/Autonomous-Mecanum-Tracking-Robot) |

Independently designed and built the complete system. The vision pipeline uses OpenCV HSV segmentation with **dual-mask fusion to handle red hue wrap-around**, contour filtering, and low-pass smoothing; positional error feeds directly into mecanum inverse kinematics for smooth omnidirectional pursuit. Pi 5 and Arduino Mega coordinate over 115200-baud serial with auto-reconnect and fail-safe stop.

</details>

<br/>

<details>
<summary><b>&nbsp;🔬 &nbsp;LSTM Autoencoder Anomaly Detection for Semiconductor Etching &nbsp;·&nbsp; Independent Research</b></summary>
<br/>

> Statistics-driven synthetic-data methodology built from **real LAM 9600 etcher data** — models trained on synthetic data only, validated on real fault wafers.

| | |
|:--|:--|
| **Stack** | Python · PyTorch · NumPy · Pandas · Statistical Modeling |
| **Scale** | Multivariate etcher telemetry; benchmarked across 5 random seeds |
| **Performance** | **75% real-fault detection vs. 25% for SPC control charts** (100% on monitored sensors) |
| **Security** | Synthetic-only training data — no proprietary fab data exposed |
| **Impact** | Outperformed Dense AE, Isolation Forest, and SPC baselines; reproducible open benchmark |
| **Repository** | [Xyllence0122/lstm-ae-anomaly-detection](https://github.com/Xyllence0122/lstm-ae-anomaly-detection) |

Designed a methodology for generating statistically faithful synthetic training data from real semiconductor equipment traces, enabling anomaly-detection research without exposing proprietary fab data. The LSTM autoencoder learns normal process signatures and flags faults by reconstruction error — rigorously benchmarked against Dense AE, Isolation Forest, and SPC.

</details>

<br/>

<details>
<summary><b>&nbsp;📐 &nbsp;Differential Drive Robot Motion Control &nbsp;·&nbsp; MATLAB / Simulink</b></summary>
<br/>

> Physics-faithful motion control simulation: **DC motor dynamics, rigid-body dynamics, and a dual-loop PID controller** converging to a circular reference trajectory from arbitrary initial poses.

| | |
|:--|:--|
| **Stack** | MATLAB · Simulink · Control Theory · Rigid-Body Dynamics |
| **Scale** | Full closed-loop simulation: motor → dynamics → kinematics → controller |
| **Performance** | Convergence to circular reference trajectory from arbitrary initial poses |
| **Security** | ±24 V actuator saturation enforced for physical realism |
| **Impact** | Resolved ±180° heading-switch oscillation via atan2 angle normalization |
| **Repository** | [github.com/Xyllence0122](https://github.com/Xyllence0122) |

A dual-loop PID architecture (position + heading) layered over modeled DC motor and rigid-body dynamics. The key engineering fix — atan2-based angle normalization — eliminated controller oscillation at the ±180° heading discontinuity, a classic failure mode in heading control.

</details>
